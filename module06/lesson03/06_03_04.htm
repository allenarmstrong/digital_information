What You See is What You Get


Imagine you are creating a web page. You finished your wireframe and storyboard, so now you are ready to begin writing code. At this stage, many web developers use a program called a WYSIWYG editor, which stands for "What You See Is What You Get" (also known as "Wizzy-Wig”). These editors show exactly what the end result of the HTML coding will look like. This is especially awesome when using images. Instead of seeing the HTML code, you can see the actual picture.




<img src=“https://pixabay.com/en/gold-medal-winner-swimming-381749/”>

Developers can explore the page layout, moving text and images around the page, before publishing the page on the web. While WYSIWYG editors are handy, they are not always accurate in their display of the code. For this reason, web developers must know HTML coding and not rely on these editors to supply code for them.
